## 学术研究场景

### 场景1.1：论文阅读与理解

**目标**: 深入理解学术论文的核心内容

**使用方法**:
```bash
# 1. 准备论文
mkdir -p data/papers
cp ~/research/*.pdf data/papers/

# 2. 启动知识库
python query_interface.py --data ./data

# 3. 添加论文
>>> /add data/papers/attention_is_all_you_need.pdf
>>> /add data/papers/bert_pretraining.pdf

# 4. 深度查询
>>> /ask 论文的核心贡献是什么？
>>> /ask 解释方法论的关键步骤
>>> /ask 实验结果的准确率是多少？
>>> /sources  # 查看数据来源
```

**最佳实践**:
- 按主题组织论文到不同文件夹
- 使用清晰的文件命名
- 定期添加新论文以扩展知识库
- 使用`/sources`命令追溯信息来源

### 场景1.2：文献综述撰写

**目标**: 基于多篇论文撰写文献综述

**使用方法**:
```bash
>>> /ask 总结这些论文中关于Transformer的研究进展
>>> /ask 对比不同方法在性能上的差异
>>> /ask 识别当前研究的空白和未来方向
>>> /ask 生成文献综述的大纲结构
```

**工作流程**:
1. 收集相关论文并添加到知识库
2. 按主题分类查询
3. 对比不同观点和方法
4. 识别研究趋势和空白
5. 生成综述大纲

## 代码开发场景

### 场景2.1：新功能开发

**目标**: 从零开始实现新功能

**使用方法**:
```bash
# 1. 分析需求
>>> /agent 分析以下需求，设计实现方案：
# 需求：为用户添加邮件通知功能

# 2. 生成代码
>>> /agent 实现邮件发送功能，使用SMTP协议
>>> /agent 添加错误处理和重试机制

# 3. 创建测试
>>> /agent 为邮件功能编写单元测试

# 4. 集成到项目
>>> /agent 将邮件功能集成到用户注册流程
```

**最佳实践**:
- 将大任务分解为小步骤
- 每个步骤完成后进行验证
- 使用`/summary`查看执行摘要
- 生成测试确保质量

### 场景2.2：代码重构

**目标**: 改善现有代码质量

**使用方法**:
```bash
# 1. 分析代码
>>> /agent 分析legacy.py，识别代码异味

# 2. 重构步骤
>>> /agent 提取重复代码为公共函数
>>> /agent 改进错误处理逻辑
>>> /agent 优化性能瓶颈

# 3. 验证重构
>>> /agent 生成测试用例验证重构结果
>>> /exec python -m pytest tests/test_legacy.py
```

### 场景2.3：Bug修复

**目标**: 定位并修复生产环境bug

**使用方法**:
```bash
# 1. 分析错误日志
>>> /agent 分析以下错误：[粘贴错误日志]
>>> /agent 可能的原因有哪些？

# 2. 定位代码
>>> /search "database error" --max-results 10
>>> /file src/database.py:200-250

# 3. 修复bug
>>> /agent 修复数据库连接泄漏问题

# 4. 验证修复
>>> /exec python -m pytest tests/test_database.py
```

## 使用建议

### 1. 场景选择建议

- **RAG模式适用**: 文档查询、知识检索、理解复杂概念
- **Agent模式适用**: 代码操作、自动化任务、复杂推理

### 2. 效率提升技巧

- 使用精确的问题描述
- 将大任务分解为小步骤
- 定期保存对话历史
- 使用`/summary`查看执行摘要

### 3. 质量保证

- 对关键代码进行人工审查
- 生成并运行测试用例
- 验证生成结果的准确性
- 使用版本控制管理代码

### 4. 安全注意事项

- 不要让Agent自动执行危险操作
- 定期备份重要文件
- 使用Git进行版本管理
- 检查生成的代码安全性
